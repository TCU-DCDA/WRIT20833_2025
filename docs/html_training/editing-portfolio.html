Editing Your Portfolio

A Beginner's Guide for WRIT 20833 Students

This guide focuses on the practical, day-to-day workflow of editing your portfolio template. If you need setup instructions, see SETUP_GUIDE.md instead.

For Beginners: Don't worry if HTML seems intimidating! This guide assumes zero experience. You'll be editing like a pro in no time.

Opening Your Project in VS Code

1 Launch VS Code

2 Open Folder (NOT File!)

Go to File ‚Üí Open Folder

3 Select Your Portfolio Folder

Navigate to wherever you cloned the repository and click Open

You should see a file tree on the left showing:
index.html, css/, images/, README.md, etc.
Pro Tip: Always open the folder, not individual files. This gives you access to all project files at once in the sidebar.

Understanding the Template Structure

File What It Is Do You Edit It?
index.html Your portfolio content ‚úÖ YES - Main file
css/styles.css Visual styling ‚ùå Only if changing colors
images/ folder Storage for charts ‚úÖ Add PNG files here
README.md Documentation ‚ùå No

Focus 95% of your time on index.html - that's where all your content goes.

The Editing Workflow

Here's the cycle you'll repeat over and over:

  1. PULL ‚Üí Get latest changes from GitHub
  2. EDIT ‚Üí Make changes in VS Code
  3. SAVE ‚Üí Save your file (Cmd+S or Ctrl+S)
  4. PREVIEW ‚Üí Check in browser
  5. COMMIT ‚Üí Save changes with GitHub Desktop
  6. PUSH ‚Üí Send changes to GitHub

üîÑ The Golden Rule

PULL ‚Üí EDIT ‚Üí COMMIT ‚Üí PUSH

Always pull before editing (especially when working in groups) to get the latest version and avoid conflicts!

Finding Your TODO Markers

The template has helpful <!-- TODO: ... --> comments showing exactly where to add content.

How to Find TODOs in VS Code

Method 1: Search

  1. Press Cmd+F (Mac) or Ctrl+F (Windows)
  2. Type TODO
  3. Press Enter to jump through each one

Method 2: TODO Tree Extension (Optional)

  1. Install "TODO Tree" extension in VS Code
  2. TODOs appear in a special sidebar panel
  3. Click to jump to each one

Example: Before and After

Before (with TODO)
<!-- TODO: Add your research question here -->
<p>What patterns emerge when we analyze [your topic]?</p>
After (your content)
<p>What patterns emerge when we analyze Taylor Swift's lyrical
   evolution using sentiment analysis and topic modeling across
   her discography?</p>
Pro Tip: Delete the TODO comment once you've completed that section. This helps you track your progress!

Editing HTML Content

Don't worry - you only need to understand a few basic patterns!

1. Paragraphs

HTML
<p>Your text goes here.</p>
You can break long paragraphs across multiple lines in your code - HTML ignores line breaks inside tags.

2. Headings

HTML
<h2>Main Section Title</h2>
<h3>Subsection Title</h3>

3. Bold and Emphasis

HTML
<p><strong>This text is bold</strong> and <em>this is italicized</em>.</p>

4. Lists

Bulleted lists:

HTML
<ul>
    <li>First point</li>
    <li>Second point</li>
    <li>Third point</li>
</ul>

Numbered lists:

HTML
<ol>
    <li>First finding</li>
    <li>Second finding</li>
    <li>Third finding</li>
</ol>

5. Links

HTML
<a href="https://github.com/yourusername/your-repo">Link text here</a>
Don't forget: Update the footer link to YOUR GitHub repository!

Editing Strategy: Copy and Modify

Keep the structure (the tags), just change the text inside:

‚ùå Don't Delete This: 
<section id="findings">
    <h2>Key Findings</h2>
‚úÖ Only Change This: 
<p>The analysis revealed...</p>

Download the PNG:

  2. Right-click your PNG file
  3. Download

2 Add to Your Project

Option A: Drag and Drop (Easiest)

  1. Open the images/ folder on your computer
  2. Drag your PNG files into that folder

Option B: Using VS Code

  1. In VS Code, right-click the images/ folder
  2. Reveal in Finder (Mac) or Reveal in File Explorer (Windows)
  3. Copy your PNG files into this folder

3 Reference in HTML

Update the image path

Common Image Path Mistakes:

  • ‚ùå src="images\chart.png" (wrong slash)
  • ‚ùå src="Downloads/chart.png" (wrong folder)
  • ‚úÖ src="images/chart.png" (CORRECT!)

Previewing Your Changes

Method 1: Open in Browser (Simplest)

  1. Save your file in VS Code (Cmd+S or Ctrl+S)
  2. Right-click index.html in VS Code
  3. Reveal in Finder/Explorer
  4. Double-click index.html
  5. Refresh browser after making more changes

Method 2: Live Server Extension (Recommended)

One-time setup:

  1. In VS Code, go to Extensions (left sidebar)
  2. Search for "Live Server"
  3. Install it

Using Live Server:

  1. Right-click anywhere in index.html
  2. Select "Open with Live Server"
  3. Browser opens automatically
  4. Auto-refreshes when you save - super convenient!
Live Server Benefit: You make a change, press Cmd+S, and instantly see the update in your browser. No manual refresh needed!

Saving with GitHub Desktop

üîÑ The Golden Rule (Again!)

PULL ‚Üí EDIT ‚Üí COMMIT ‚Üí PUSH

Always in this order!

1 PULL First (Critical!)

  1. Open GitHub Desktop
  2. Click "Fetch origin" (top right)
  3. If it says "Pull origin", click that too
Why pull first? This gets any changes your teammates made. Always pull BEFORE editing to avoid merge conflicts.

2 Make Your Edits in VS Code

Edit index.html, add images, etc.

3 Check What Changed

Go back to GitHub Desktop. You'll see:

  • Left panel: Files you changed
  • Right panel: Exactly what changed (red = deleted, green = added)

4 Commit Your Changes

At the bottom left of GitHub Desktop:

Good commit message example
Summary: Added research question and data description

Description:
- Wrote my research question section
- Added data source and ethics discussion
- Still need to add visualizations

Click "Commit to main"

5 PUSH to GitHub

Click "Push origin" (top right button)

Done! Your changes are now on GitHub and visible to your team.

Good vs. Bad Commit Messages

‚úÖ Good:
  • "Added sentiment analysis section with code examples"
  • "Fixed broken image links"
  • "Updated research question"
‚ùå Not Helpful:
  • "updated stuff"
  • "changes"
  • "asdfjkl"

Common Beginner Mistakes

1. Forgetting to Save

Problem: Made changes, refreshed browser, nothing happened.

Solution: Press Cmd+S or Ctrl+S. Look for the white dot next to the filename in VS Code - if you see it, the file isn't saved.

2. Opening Individual Files Instead of Folder

Problem: Can't see other project files in VS Code.

Solution: File ‚Üí Open Folder, then select your portfolio folder. NOT "Open File"!

3. Wrong Image Paths

Problem: Images show as broken icons.

‚ùå Wrong:
  • Images/ or img/
  • C:/Users/Downloads/
  • Wrong filename case
‚úÖ Correct:
  • images/filename.png
  • Exact filename match
  • Forward slashes

4. Forgetting to Pull Before Editing (Groups)

Problem: "Merge conflict" error when trying to push.

Solution:

  • ALWAYS click "Fetch origin" ‚Üí "Pull" before editing
  • Communicate with team: "I'm editing Methods now"
  • Wait for teammates to push before you start

5. Deleting HTML Tags Accidentally

Problem: Page looks broken, styling disappeared.

Solution:

  • Only edit text inside tags, not the tags themselves
  • If you deleted </section> by accident, undo (Cmd+Z)
  • Use VS Code's auto-indent: Select all (Cmd+A), then Format Document

6. Not Testing Before Committing

Problem: Pushed broken code to GitHub, entire team affected.

Solution:

  1. Always preview in browser before committing
  2. Check that images load
  3. Read through for typos
  4. THEN commit and push

Quick Troubleshooting

Images Not Showing

  • File is in images/ folder (not Images/, not root)
  • Filename in HTML matches exactly (case-sensitive)
  • Path uses forward slash: images/chart.png
  • File is actually a PNG (not .pdf, .ai, etc.)
  • You saved the HTML file
  • You hard-refreshed browser (Cmd+Shift+R or Ctrl+Shift+R)

CSS Not Loading (Page Looks Plain)

  • styles.css is in css/ folder
  • Link tag in HTML: <link rel="stylesheet" href="css/styles.css">
  • Hard refresh browser (Cmd+Shift+R or Ctrl+Shift+R)

Can't Push to GitHub

Error: "You don't have permission"

  • If in a group: Repository owner must add you as collaborator
  • Check you're logged into correct GitHub account in GitHub Desktop

Error: "Need to pull first"

  1. Click "Fetch origin"
  2. Click "Pull origin"
  3. If conflicts appear, see Merge Conflicts section below

Merge Conflicts (Group Work)

What it means: You and a teammate edited the same lines.

How to fix:

  1. GitHub Desktop shows which file has conflicts
  2. Open that file in VS Code
  3. Look for conflict markers: 
    <<<<<<< HEAD
Your version here
=======
Teammate's version here
>>>>>>> main
  4. Delete the markers and keep the version you want (or combine both)
  5. Save the file
  6. Go to GitHub Desktop ‚Üí Commit ‚Üí Push
Prevention: Coordinate with your team - work on different sections at different times! Communicate constantly.

Working as a Team: Best Practices

Communication is Everything

Before editing:
"Hey team, I'm going to work on the Data & Methods section for the next hour. Don't edit that part!"
After editing:
"Methods section is done and pushed. Safe to pull now!"

Divide Sections Clearly

Example for a 3-person team:

  • Person A: Research Question + Background
  • Person B: Data & Methods + Analysis code
  • Person C: Findings + Reflection + Images

Everyone: Pull before editing, push when done, communicate constantly!
