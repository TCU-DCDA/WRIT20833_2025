Common Pitfalls & Solutions

‚ùå Problem

"My changes aren't showing on the live site"

‚úÖ Solution:

  • Check commit history‚Äîdid you push?
  • Wait 1-2 minutes for Pages rebuild
  • Hard refresh browser (Cmd+Shift+R)
  • Check Pages settings are correct

‚ùå Problem

"I got merge conflicts"

‚úÖ Solution:

  • Always "Fetch origin" before making changes
  • Don't edit files directly on GitHub.com
  • If conflicts occur, GitHub Desktop will alert you
  • Open the conflicted file in your editor and choose which version to keep

‚ùå Problem

"My site shows 404 error"

‚úÖ Solution:

  • Ensure index.html is in correct folder
  • Check Pages source settings
  • Verify repository name in URL
  • Make sure repository is public

‚ùå Problem

"I accidentally committed sensitive data"

‚úÖ Solution:

  • Don't panic‚Äîbut act quickly
  • Delete the file, commit, push
  • Change any exposed passwords immediately
  • For thorough removal, use BFG Repo-Cleaner

Best Practices

‚úÖ Do This

  • Commit often with clear messages
    "Add navigation bar" not "update"
  • Use README.md to document your project
  • Organize files in folders (images/, css/, js/)
  • Test locally in browser before committing
  • Fetch origin regularly to stay in sync
  • Review changes in GitHub Desktop before committing

‚ùå Don't Do This

  • Never commit passwords or API keys
  • Don't use vague commit messages
    "fixed stuff", "asdf"
  • Avoid committing large files (>100MB)
  • Don't edit directly on GitHub.com (use local editor)
  • Don't commit personal information
  • Avoid messy file structures

Remember: Your GitHub repository is both your backup and your public portfolio‚Äîtreat it professionally
