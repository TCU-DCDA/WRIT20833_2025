HTML Document Anatomy

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Topic Modeling Results</title>
  </head>
  <body>
    <!-- Your visible content goes here -->
  </body>
</html>

<head>

Metadata - information about your page (title, description, links to CSS)

<body>

Content - everything users actually see and interact with

Think of it like a research paper: <head> = title page/metadata, <body> = actual content

Semantic HTML5 Elements

<header>
Site/page introduction
<nav>
Navigation links
<main>
Primary content
<article>
Standalone content
<aside>
Related/supplementary
<footer>
Site/page conclusion

Why Use Semantic Tags?

  • Meaning: Tags communicate purpose to humans & machines
  • Accessibility: Screen readers understand document structure
  • Argument: Structure = claim about what's important

Headings Create Hierarchy

<h1>Sentiment Analysis of 
Climate Discourse</h1>

<h2>Methods</h2>
  <h3>Data Collection</h3>
  <h3>VADER Analysis</h3>

<h2>Results</h2>
  <h3>Overall Trends</h3>
  <h3>Key Patterns</h3>

Heading Rules:

  • <h1>: One per page - main topic
  • <h2>: Major sections
  • <h3>: Subsections within h2
  • Don't skip levels (h1 → h3)

Rhetorical Impact:

Heading hierarchy = argument about relative importance. What deserves h2 vs h3 is a rhetorical choice.

DH Example: Digital Archive

Community Photo Archive - Two Structural Approaches:

Approach A: Academic Frame

<header>
  <h1>Research Project</h1>
</header>
<main>
  <section>
    <h2>Methodology</h2>
    <article>...</article>
  </section>
  <section>
    <h2>Gallery</h2>
  </section>
</main>
<aside>Community</aside>

Argument: Research methodology is primary. Photos are evidence. Community input is supplementary.

Approach B: Community-Centered

<header>
  <h1>Our Community</h1>
</header>
<main>
  <section>
    <h2>Photo Stories</h2>
    <article>...</article>
  </section>
</main>
<aside>Research</aside>

Argument: Community voices and photos are primary. Research context is supplementary.

Lesson: Same content, different HTML structure = different rhetorical stance about whose knowledge matters.

Links as Pathways

Link Anatomy:

<a href="results.html">
  View Analysis Results
</a>

<a href="https://example.com">
  External Resource
</a>

Best Practices:

  • Descriptive text: "View sentiment analysis results" not "click here"
  • Context: Tell users where they're going
  • Accessibility: Screen readers read link text out of context

Navigation Structure:

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="findings.html">Findings</a></li>
    <li><a href="sources.html">Sources</a></li>
  </ul>
</nav>

Rhetorical Question:

What you include in <nav> = "these are the important destinations." What paths do you prioritize?

Images & Accessibility

❌ Inaccessible:

<img src="chart.jpg">

Screen reader: "Image" (no information)

Excludes: Users who can't see images

⚠️ Minimal:

<img src="chart.jpg" 
     alt="chart">

Screen reader: "Chart" (not helpful)

Partially excludes: No actual content

✓ Accessible:

<img src="chart.jpg"
     alt="Bar chart showing 
     60% increase in negative 
     sentiment about climate 
     policy from 2015 to 2020">

Screen reader: Full insight available

Includes: Everyone gets the data

Framework: Sacred Boundaries - Alt text is an ethical decision about who deserves access to your insights.

Lists for Organization

Unordered Lists (bullets):

<ul>
  <li>Topic: Politics</li>
  <li>Topic: Technology</li>
  <li>Topic: Health</li>
</ul>

Use when: Order doesn't matter (topics, features, themes)

Ordered Lists (numbered):

<ol>
  <li>Collect data</li>
  <li>Clean and preprocess</li>
  <li>Run topic modeling</li>
  <li>Interpret results</li>
</ol>

Use when: Order matters (steps, rankings, chronology)

Remember: Choosing ul vs ol = rhetorical choice about whether sequence matters.

Full DH Page Structure

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Climate Discourse Sentiment Analysis</title>
</head>
<body>
  <header>
    <h1>Climate Discourse Sentiment Analysis</h1>
    <p>Tracking emotional tone in climate 
    conversations, 2015-2020</p>
  </header>

  <nav>
    <ul>
      <li><a href="#methods">Methods</a></li>
      <li><a href="#findings">Findings</a></li>
      <li><a href="#implications">Implications</a></li>
    </ul>
  </nav>

  <main>
    <section id="methods">
      <h2>Methods</h2>
      <p>We analyzed 50,000 tweets using 
      VADER sentiment analysis...</p>
      <img src="methodology.jpg" 
           alt="Flowchart showing data collection
           from Twitter API, preprocessing steps, 
           and VADER analysis">
    </section>

    <section id="findings">
      <h2>Key Findings</h2>
      <article>
        <h3>Negative Sentiment Increased 60%</h3>
        <p>From 2015 to 2020, negative sentiment 
        rose sharply...</p>
        <img src="trend.jpg" 
             alt="Line graph showing negative 
             sentiment rising from 20% in 2015 
             to 80% in 2020">
      </article>
    </section>
  </main>

  <aside>
    <h3>Data Sources</h3>
    <ul>
      <li>Twitter API (2015-2020)</li>
      <li>50,000 tweets with #climate</li>
    </ul>
  </aside>

  <footer>
    <p>Created by [Your Name] | WRIT 20833 | 
    Fall 2025</p>
  </footer>
</body>
</html>

Notice: Semantic structure tells a story about what matters and how pieces relate.
