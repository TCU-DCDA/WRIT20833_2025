## General Style Guidelines

**Color Palette (consistent across all images):**
- Primary Purple: #9b59b6 (code/technical)
- Light Purple: #c9a9ff
- Blue: #3498db (accessibility/ethics)
- Green: #2ecc71 (inclusive/good practice)
- Red: #ff6b6b (exclusion/problems)
- Yellow/Amber: #f1c40f (rhetoric/emphasis)
- Dark Background: #1a0330 to #2d1b5e

**Aesthetic Principles:**
- Conceptual but clear
- Shows code as argumentative, not neutral
- Reveals hidden rhetorical dimensions
- Critical and questioning
- Connects technical to ethical
- Shows power dynamics in structure
- Emphasizes choice and consequence
- Bridge between code and rhetoric

**Thematic Emphasis:**
- Every technical choice is rhetorical
- Structure = argument about importance
- Styling = claims about value
- Access = ethical stance
- Code makes invisible arguments visible
- Neutrality is a myth
- Awareness enables critical practice

**Technical Requirements:**
- **Slides 1, 3, 7:** Landscape orientation (16:9), size 1200x675px
- **Slides 2, 4, 5, 6, 8:** Portrait orientation (3:4), size 800x1200px
- JPEG format for web optimization (target <200KB)
- sRGB color space for web
- High contrast for readability
- Clear conceptual metaphors
- Shows transformation/revelation
- Annotated when helpful

## Bridge Lecture Visual Strategy

This is a **conceptual bridge** lecture, so visuals should:
1. **Reveal hidden dimensions** - make invisible rhetoric visible
2. **Show transformation** - from neutral to argumentative framing
3. **Use metaphor effectively** - abstract concepts need visual anchors
4. **Connect technical to humanistic** - bridge two ways of thinking
5. **Prepare for instruction** - set up critical mindset before HTML/CSS teaching

Color coding:
- **Purple** = code/technical domain
- **Amber** = rhetoric/argument domain
- **Red** = problems/exclusions
- **Green** = inclusive/ethical choices
- **Blue** = access/ethics
