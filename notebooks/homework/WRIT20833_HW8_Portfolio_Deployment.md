## Troubleshooting Common Issues

### Issue: Site shows "404 Page Not Found"
**Solutions**:
- Wait 2-3 minutes after enabling Pages (deployment takes time)
- Check that main branch is selected in Pages settings
- Verify repository is public
- Ensure `index.html` exists (GitHub Pages needs this as homepage)

### Issue: Images don't load on live site
**Solutions**:
- Check image file paths use relative paths: `images/chart.png`
- Verify image filenames match exactly (case-sensitive!)
- Confirm images exist in repository and were pushed to GitHub
- Check `images/` folder is committed and pushed

### Issue: CSS styling doesn't appear
**Solutions**:
- Verify `<link rel="stylesheet" href="style.css">` exists in all HTML files
- Check `style.css` filename matches exactly (case-sensitive)
- Confirm `style.css` is committed and pushed to GitHub
- Clear browser cache and refresh

### Issue: Navigation links broken
**Solutions**:
- Use relative links: `<a href="data.html">` not `<a href="/data.html">`
- Verify HTML filenames match link hrefs exactly
- Test links locally in Live Server before deploying

### Issue: Site works locally but not on GitHub Pages
**Solutions**:
- This usually indicates file path issues
- Use only relative paths (no leading `/`)
- Check for case sensitivity in filenames
- Commit and push all files to GitHub

### Deployment Tips
- **Test thoroughly locally first**: Fix all issues before deploying
